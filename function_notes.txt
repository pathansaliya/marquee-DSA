# Java Functions, Memory & OOP Notes

# 1. What is a Function (Method) in Java?

A function (called a method in Java) is a block of code used to perform a specific task.

## Example

```java
class Demo {

    void greet() {
        System.out.println("Hello");
    }
}
```

---

# 2. Why Do We Use Functions?

Functions are used to:

* Reduce code repetition
* Improve readability
* Reuse code
* Make debugging easier
* Divide large programs into smaller parts

---

# 3. Components of a Function

## Syntax

```java
accessModifier returnType methodName(parameters) {
    // body
}
```

## Example

```java
public int add(int a, int b) {
    return a + b;
}
```

## Components Explanation

| Component    | Meaning         |
| ------------ | --------------- |
| public       | Access modifier |
| int          | Return type     |
| add          | Method name     |
| int a, int b | Parameters      |
| return       | Returns value   |

---

# 4. Parameterized and Non-Parameterized Functions

## Non-Parameterized Function

A function without parameters.

```java
void show() {
    System.out.println("Hello");
}
```

## Parameterized Function

A function with parameters.

```java
void show(int a) {
    System.out.println(a);
}
```

---

# 5. Parameters vs Arguments

| Parameters                       | Arguments            |
| -------------------------------- | -------------------- |
| Variables in function definition | Actual values passed |
| Placeholder values               | Real values          |
| int a                            | 10                   |

## Example

```java
void sum(int a, int b) {
    System.out.println(a + b);
}

sum(5, 10);
```

Here:

* `a` and `b` are parameters
* `5` and `10` are arguments

---

# 6. Sequential Flow of Function Calls

Program execution follows this order:

1. main() starts
2. Function call happens
3. Control moves to called function
4. Function executes
5. Control returns back

## Example

```java
class Demo {

    static void fun() {
        System.out.println("Function");
    }

    public static void main(String[] args) {

        System.out.println("Start");

        fun();

        System.out.println("End");
    }
}
```

## Output

```java
Start
Function
End
```

# 7. Scope in Java

# (a) Local Scope

Variables declared inside methods or blocks.

```java
void show() {
    int a = 5;
}
```

`a` can only be used inside `show()`.

---

# (b) Global Scope (Class Scope)

Variables declared inside class but outside methods.

```java
class Demo {
    int x = 10;
}
```

`x` can be accessed by all methods of the class.

---

# (c) Lexical Scope

Accessibility depends on physical location of variables.

Inner blocks can access outer variables.

```java
int a = 10;

{
    System.out.println(a);
}
```

---

# 8. this Keyword

`this` refers to the current object.

## Example

```java
class Student {

    int age;

    Student(int age) {
        this.age = age;
    }
}
```

`this.age` refers to class variable.

---

# 9. Stack Memory

Stack memory stores:

* Method calls
* Local variables
* Function stack frames

## Features

* Fast memory
* Temporary memory
* Works on LIFO (Last In First Out)

## Example

```java
void show() {
    int a = 10;
}
```

`a` is stored in stack memory.

---

# 10. Heap Memory

Heap memory stores:

* Objects
* Instance variables
* Strings

## Features

* Persistent memory
* Slower than stack
* Shared memory

## Example

```java
Student s = new Student();
```

Object is created inside heap memory.

---

# 11. Stack Frame

Every function call creates its own memory area in stack memory called a stack frame.

It stores:

* Local variables
* Parameters
* Return address

## Example

```java
fun1();
fun2();
fun3();
```

## Stack Order

```text
fun3()
fun2()
fun1()
main()
```

Last function entered exits first.

---

# 12. Object Storage in Memory

## Example

```java
Student s = new Student();
```

| Part                     | Memory |
| ------------------------ | ------ |
| s reference variable     | Stack  |
| Object created using new | Heap   |

# 15. Pass By Value and Pass By Reference

# Pass By Value

Copy of value is passed.

Used with primitive data types.

```java
int a = 5;
```

Changes do not affect original value.

## Example

```java
class Demo {

    static void change(int x) {
        x = 20;
    }

    public static void main(String[] args) {

        int a = 10;

        change(a);

        System.out.println(a);
    }
}
```

## Output

```java
10
```

---

# Pass By Reference Behavior

Reference of object is passed.

```java
Student s = new Student();
```

Object data can change using reference.

## Example

```java
class Student {
    int age = 18;
}

class Demo {

    static void change(Student s) {
        s.age = 25;
    }

    public static void main(String[] args) {

        Student s1 = new Student();

        change(s1);

        System.out.println(s1.age);
    }
}
```

## Output

```java
25
```

---

# 16. Recursive Function

A function calling itself is called recursion.

## Example

```java
void fun() {
    fun();
}
```

---

# Factorial Using Recursion

```java
class Demo {

    static int fact(int n) {

        if(n == 1)
            return 1;

        return n * fact(n - 1);
    }

    public static void main(String[] args) {

        System.out.println(fact(5));
    }
}
```

## Output

```java
120
```

---

# 17. Variable Shadowing / Overshadowing

When local variable and class variable have same name, local variable gets priority.

## Example

```java
class Demo {

    int x = 10;

    void show() {

        int x = 20;

        System.out.println(x);
    }
}
```

## Output

```java
20
```

To access class variable:

```java
this.x
```

---

# 18. Function Overloading

Function overloading means same method name with different parameters.

## Rules

* Parameters must be different
* Return type may or may not be different
* Only changing return type is NOT enough

## Example

```java
class Demo {

    void show(int a) {
        System.out.println(a);
    }

    void show(String s) {
        System.out.println(s);
    }
}
```

---

# 19. Method Overriding

Method overriding means child class provides different implementation of parent method.

## Rules

* Same method name
* Same parameters
* Different implementation

## Example

```java
class Parent {

    void show() {
        System.out.println("Parent");
    }
}

class Child extends Parent {

    void show() {
        System.out.println("Child");
    }
}
```

---

# 20. Inheritance

Inheritance means child class inherits properties and methods of parent class.

## Syntax

```java
class Child extends Parent {

}
```

## Example

```java
class Animal {

    void eat() {
        System.out.println("Eating");
    }
}

class Dog extends Animal {

    void bark() {
        System.out.println("Barking");
    }
}
```

## Benefits

* Code reuse
* Better organization
* Supports overriding

---

# 21. Important Quick Points

* Stack memory is faster than heap memory
* Heap memory stores objects until garbage collection
* Every function call creates a stack frame
* Java follows LIFO in function calls
* Recursion heavily uses stack memory
* Static and final methods cannot be overridden
* Overloading = Compile-time polymorphism
* Overriding = Runtime polymorphism
